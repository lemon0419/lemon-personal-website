# 需求文档

## 1. 应用概述

**应用名称**：张巧楠个人网站

**应用描述**：一个简约清新的个人主页网站，展示张巧楠的个人信息，并提供数字分身AI聊天功能，供朋友、对其感兴趣的人以及HR访问了解。

## 2. 用户与使用场景

**目标用户**：
- 张巧楠的朋友
- 对张巧楠感兴趣的人
- HR（招聘方）

**核心使用场景**：
- 访客通过网站了解张巧楠的基本信息、兴趣爱好和近期动态
- 访客通过数字分身聊天区与「张巧楠数字分身」进行对话，获取更多信息
- 访客根据个人偏好切换中英文版本和日夜模式

## 3. 页面结构与功能说明

### 页面层级结构

```
张巧楠个人网站
├── 开场动画页
└── 主页面
    ├── 头部区域
    ├── 关于我区域
    ├── 个人信息展示区
    └── 数字分身聊天区
```

### 3.1 开场动画页

**功能说明**：
- 页面展示一个插画小人形象（代表张巧楠），配合欢迎文案「欢迎大家来参观我的个人网页」
- 访客点击页面任意位置或插画小人后，触发动画效果：插画小人做拉开幕布的动作
- 动画结束后自动进入主页面

**视觉风格**：
- 插画风格：线条插画风格、手绘感
- 颜色：莫兰迪色系
- 留白充分，文艺清新

### 3.2 主页面

#### 3.2.1 头部区域

**功能说明**：
- 展示网站标题或Logo
- 提供中英文切换按钮
- 提供日夜模式切换按钮

**交互说明**：
- 点击中英文切换按钮，页面所有文本内容在中文和英文之间切换
- 点击日夜模式切换按钮，页面配色在日间模式和夜间模式之间切换

#### 3.2.2 关于我区域

**功能说明**：
- 展示头像（插画风格）
- 展示名字：张巧楠
- 展示一句话介绍：一个普通人，但想对世界产生一些影响

#### 3.2.3 个人信息展示区

**功能说明**：
- 展示以下个人信息：
  - 我现在主要在做：具体没有在做什么，处于一个学习的阶段
  - 我的兴趣：我喜欢看小说、电视剧，喜欢走进剧院，最喜欢听音乐剧，其次是话剧和舞剧
  - 我一个比较有记忆点的特点：随性、没有什么特定的目的地

**视觉风格**：
- 信息层次清晰，采用简约布局
- 使用手绘文字感的字体或装饰元素
- 留白充分

#### 3.2.4 数字分身聊天区

**功能说明**：
- 提供聊天输入框，访客可输入问题
- 访客提交问题后，调用文心大模型API，基于预设的张巧楠个人信息生成回答
- 数字分身以第一人称回答访客问题
- 展示聊天记录（访客问题和数字分身回答）

**数字分身预设信息**：
- 职业或身份：产品运营
- 最近在做的事：搭自己的个人主页，整理作品集、学习ai的应用
- 擅长或关心的方向：女性主义、内容表达、知识整理
- 常见问题参考：
  1. 你现在在做什么？
  2. 你有哪些作品？
  3. 怎么联系你？

**交互说明**：
- 访客在输入框输入问题，点击发送按钮或按回车键提交
- 提交后，聊天区显示访客问题
- 调用文心大模型API，获取回答后，聊天区显示数字分身回答
- 聊天记录按时间顺序排列，支持滚动查看历史记录

## 4. 业务规则与逻辑

### 4.1 中英文切换规则

- 默认语言：中文
- 切换范围：页面所有文本内容（包括关于我区域、个人信息展示区、数字分身聊天区的界面文案）
- 切换后状态保持：访客切换语言后，刷新页面或重新访问时，保持上次选择的语言

### 4.2 日夜模式切换规则

- 默认模式：日间模式
- 切换范围：页面整体配色方案
- 日间模式：浅色背景，深色文字，莫兰迪色系
- 夜间模式：深色背景，浅色文字，莫兰迪色系
- 切换后状态保持：访客切换模式后，刷新页面或重新访问时，保持上次选择的模式

### 4.3 数字分身对话规则

- 数字分身基于预设的张巧楠个人信息回答问题
- 回答使用第一人称（我）
- 调用文心大模型API实现AI对话能力
- 若API调用失败，聊天区显示错误提示

### 4.4 移动端适配规则

- 网站需适配手机屏幕尺寸
- 页面布局在移动端自动调整为单列布局
- 交互元素（按钮、输入框）在移动端保持可点击和可操作

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 访客在数字分身聊天区输入空内容 | 禁止提交，提示「请输入问题」 |
| 文心大模型API调用失败 | 聊天区显示「抱歉，暂时无法回答，请稍后再试」 |
| 文心大模型API响应超时 | 聊天区显示「回答生成中，请稍候」，超过30秒后显示「响应超时，请重试」 |
| 访客在开场动画页未点击，停留超过10秒 | 自动播放拉开幕布动画，进入主页面 |
| 访客刷新页面 | 保持上次选择的语言和日夜模式 |

## 6. 验收标准

1. 访客打开网站，看到开场动画页，展示插画小人和欢迎文案
2. 访客点击页面，插画小人做拉开幕布动作，进入主页面
3. 访客在主页面看到头像、名字、一句话介绍、个人信息展示区和数字分身聊天区
4. 访客点击中英文切换按钮，页面文本内容在中文和英文之间切换
5. 访客点击日夜模式切换按钮，页面配色在日间模式和夜间模式之间切换
6. 访客在数字分身聊天区输入问题「你现在在做什么？」，点击发送
7. 数字分身以第一人称回答问题，聊天区显示访客问题和数字分身回答
8. 访客在手机端打开网站，页面布局自动适配手机屏幕，所有功能正常使用
